# ADR-004 — Optimistic Locking para Reserva de Citas

## Estado
Propuesto

## Contexto
En `scheduling-service`, múltiples recepcionistas o procesos pueden intentar
reservar el mismo slot (`physician_id`, `slot_start`) casi al mismo tiempo.
Sin control de concurrencia, esto puede provocar double booking.

Adicionalmente, el sistema requiere devolver al cliente una respuesta de
conflicto con exactamente 3 alternativas de horario cuando la reserva no se
puede materializar.

## Decisión
Se adopta un enfoque de concurrencia optimista con tres mecanismos
complementarios:

1. Campo `version` en la entidad JPA de citas (`@Version`) para detectar
   actualizaciones concurrentes.
2. Restricción de unicidad en persistencia para el slot del médico
   (`physician_id`, `slot_start`) para impedir doble inserción efectiva.
3. Manejo de conflicto en aplicación (`BookAppointmentUseCase`) capturando
   excepciones de concurrencia/constraint y mapeando a
   `SlotAlreadyBookedException` con 3 alternativas calculadas.

## Alternativas consideradas
- Locking pesimista (`SELECT ... FOR UPDATE`) — descartada porque incrementa
  contención y latencia bajo carga, bloqueando más tiempo transacciones.
- Serialización global por cola de reservas — descartada porque añade
  complejidad operativa (infra, retries, orden global) para un caso de uso
  que puede resolverse con garantías de base de datos + capa de aplicación.

## Consecuencias
- Positivas:
  - Evita double booking de manera robusta incluso con concurrencia alta.
  - Mantiene buen throughput al no bloquear pesimistamente por defecto.
  - Permite respuesta de negocio clara (`409`) con alternativas.
- Negativas / trade-offs:
  - Puede aumentar tasa de conflictos/reintentos en horas pico.
  - Requiere disciplina de mapeo consistente de excepciones técnicas a errores
    de dominio.

## Trigger de revisión
Reevaluar esta decisión si:
- la tasa de conflictos supera el umbral operativo aceptable en producción,
- aparecen transacciones largas que incrementen reintentos,
- se requieren garantías de orden estricto entre múltiples nodos de escritura.
